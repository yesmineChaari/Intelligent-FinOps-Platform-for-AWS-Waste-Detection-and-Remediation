PROJECT SUMMARY (LLM CONTEXT)

1) High-level purpose
This repository implements Phase 1 of a FinOps pipeline ("Agent 2").
It analyzes EC2 instances using statistically grounded utilization metrics and emits in-memory recommendations for actions such as TERMINATE, STOP, DOWNSIZE, SKIP, or CLEAN.
Phase 1 is read-only against the database: it queries metadata, metrics, and pricing, computes results, logs JSON output, and returns objects to the next phase.

2) Top-level structure
- .env: runtime configuration (loaded with python-dotenv).
- main.py: async entrypoint/orchestrator for Agent 2.
- requirements.txt: pinned Python dependencies.
- rules.yaml: threshold/configuration policy for detection and sizing.
- phase1/: package containing models, loader, DB queries, detection logic, and sizing algorithm.
  - phase1/init.py: package exports.
  - phase1/models.py: Pydantic schemas and enums.
  - phase1/loader.py: YAML load + validation into typed Rules model.
  - phase1/queries.py: all SQL access functions.
  - phase1/detection.py: role-based detection pipeline and result assembly.
  - phase1/sizing.py: recommended downsize selection logic.

3) Core libraries and why they are used
- asyncio: asynchronous orchestration of IO-bound steps.
- asyncpg: async PostgreSQL client (Neon DB connection).
- redis.asyncio (redis): async Redis stream trigger consumption.
- pydantic: strict validation of rules and output models.
- PyYAML (yaml): reading rules.yaml.
- python-dotenv: loading environment variables from .env.
- stdlib: os/json/logging/sys/datetime/pathlib/enum/typing.

4) Runtime pipeline (end-to-end)
A. Startup (main.py)
- load_dotenv() reads .env.
- Required env var: NEON_DATABASE_URL.
- Optional env vars:
  - REDIS_URL (default redis://localhost:6379)
  - RULES_PATH (default rules.yaml)
  - SKIP_REDIS_TRIGGER (if 1/true/yes, skip waiting for Redis event)
- rules.yaml is loaded and validated once at startup (fail-fast if invalid).

B. Trigger handling
- If SKIP_REDIS_TRIGGER is not set, waits indefinitely on Redis stream "ingestion_stream" for event=ingestion_complete via XREAD.
- If SKIP_REDIS_TRIGGER is set, Phase 1 runs immediately.

C. Phase 1 processing
- Connect to Neon DB using asyncpg.
- run_phase1(conn, rules) is called.
- All EC2 resources are pulled from resources table.
- For each instance:
  1) Global zombie check first (status/age based) regardless of role.
  2) Skip if role in detection.skipped_roles.
  3) Route by role:
     - dependent_primary -> dependent_primary logic.
     - bursty -> bursty logic.
     - everything else -> steady logic.

D. Decision output
- For each instance, returns Phase1Result with:
  - action + waste_type,
  - supporting metrics (where applicable),
  - optional sizing recommendation/cost deltas,
  - human-readable detection_reason.
- Results are logged and printed as JSON (exclude_none=True) and returned in-memory.
- Connections are closed in finally block.

5) Files and key functions

main.py
- wait_for_trigger(redis_client): blocks on Redis stream until ingestion_complete.
- main(): environment setup, rules load, optional trigger wait, DB connect, run_phase1, output/logging, cleanup.

phase1/models.py
- Enums:
  - WasteAction: TERMINATE, STOP, DOWNSIZE, SKIP, CLEAN.
  - WasteType: zombie, idle, oversized, tag_error, none.
- Output model:
  - Phase1Result: canonical output contract including metrics and sizing fields.
- Config models:
  - Rules root object, with nested DetectionRules, SizingRules, and role-specific rule models.

phase1/loader.py
- load_rules(path="rules.yaml"):
  - Ensures rules file exists.
  - Loads YAML.
  - Validates into Rules via Pydantic.
  - Raises on missing file or schema/type mismatch.

phase1/queries.py
- get_all_instances(conn): fetches EC2 rows from resources.
- get_instance_metrics(conn, instance_id, window_days): computes:
  - P95 CPU, P99 CPU, Max CPU, P95 RAM, CV (stddev/mean).
- is_zombie(conn, instance_id, stopped_days_threshold): true if stopped and old enough.
- get_sizing_ladder(conn, instance_family, region, os_type): pricing ladder for family/region/os ordered by ladder_rank.
- get_instance_price(conn, instance_type, region, os_type): specific hourly price lookup.

phase1/detection.py
- run_phase1(conn, rules): loops instances and collects Phase1Result.
- _process_instance(...): orchestration per instance (zombie -> skip -> role routing).
- _detect_dependent_primary(...): idle check using p95 thresholds; may downsize.
- _detect_bursty(...): validates CV; if truly bursty and p99 below threshold, marks oversized + sizing.
- _detect_steady(...): two-step steady analysis:
  - idle check (p95 cpu/ram + max cpu) -> STOP,
  - oversized check (p95 cpu/ram) -> DOWNSIZE + sizing.
- _get_sizing(...): gathers ladder and current entry, delegates recommendation algorithm.
- _flatten_sizing(...): maps sizing recommendation into output payload fields.
- _skip(...) and _clean(...): helper result constructors.

phase1/sizing.py
- calculate_recommended_type(...):
  - Derives consumed RAM from observed %.
  - Finds lower-ranked candidates in same family ladder.
  - Applies max_drop_steps guard.
  - Projects CPU utilization scaling by vCPU ratio.
  - Projects RAM utilization on candidate RAM capacity.
  - Enforces ceilings/headroom:
    - projected CPU must stay below cpu_safety_ceiling.
    - projected RAM must stay <= ram_headroom_threshold.
  - Returns best valid candidate (or None) with projected metrics and monthly waste savings.

phase1/init.py
- Re-exports load_rules and run_phase1.

6) Config contract (rules.yaml)
- detection.skipped_roles: roles bypassed with SKIP action.
- detection.zombie: termination rule for long-stopped instances.
- detection.dependent_primary: idle thresholds and forced action.
- detection.bursty: CV threshold + p99 threshold.
- detection.steady.idle: strict dead-idle thresholds.
- detection.steady.oversized: moderate utilization thresholds for downsizing.
- sizing:
  - max_drop_steps,
  - ram_headroom_threshold,
  - cpu_safety_ceiling.

7) Data/table assumptions inferred from SQL
- resources table fields used:
  instance_id, instance_type, status, launched_at, role, os, region, environment, team, resource_type.
- ec2_metrics table fields used:
  instance_id, timestamp, cpu_pct, ram_pct.
- pricing table fields used:
  instance_type, instance_family, region, os, ladder_rank, vcpu, ram_gb, price_per_hour.

8) Operational characteristics
- Asynchronous, IO-bound pipeline.
- Stateless between runs except external DB/Redis state.
- Read-only on DB in this phase.
- Designed to fail fast on bad configuration.
- Produces typed, explainable outputs suitable for downstream automation.

9) Current status note
- The earlier sizing invocation bug (passing a string where dict was expected) has been corrected by passing the full instance object into _get_sizing calls.

10) How to run
- Activate env and install dependencies from requirements.txt.
- Ensure .env exists with at least NEON_DATABASE_URL.
- Optionally set SKIP_REDIS_TRIGGER=1 for local immediate execution.
- Run: python main.py
